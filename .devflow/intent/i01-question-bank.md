## 1. Original Request

> Build a small Quiz Bank web application as the official example project for DevFlow.

The example should demonstrate how DevFlow transforms a user's original request into an interview, objective, executable tasks, evidence, and project status.

The application should remain intentionally small and focus on demonstrating the DevFlow workflow rather than building a production-ready system.

---

## 2. Motivation

The Quiz Bank project serves as the reference project for DevFlow.

Its primary purpose is to demonstrate:

* How an original request evolves through the DevFlow workflow.
* How AI-assisted development can be resumed from project artifacts instead of chat history.
* How each development iteration produces explicit and traceable artifacts.

The project should be simple enough for new users to understand while remaining realistic enough to demonstrate a complete software development workflow.

---

## 3. Known Constraints

* This is a demonstration project, not a commercial product.
* The implementation should remain intentionally lightweight.
* The application should use the approved UI design.
* The project should follow the DevFlow workflow.
* Future functionality will be introduced through additional development iterations.
* The UI architecture should remain stable as new functionality is added.
* Every development iteration should begin with a new Intent.
